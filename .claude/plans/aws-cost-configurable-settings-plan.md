# Plan: Make AWS Cost settings configurable

## Goal
Expose four AWS Cost connector knobs in the Settings → Connectors UI so users can override the auto-estimated budget, set delta/risk thresholds, and control how many cost drivers are surfaced.

## Settings to add
1. **Monthly budget override** — replaces the auto-estimated "last month + 10%" budget.
2. **Cost increase alert threshold (%)** — the month-over-month delta % that creates a `cost_driver` event. Currently hard-coded to 25%.
3. **Critical cost risk threshold (%)** — the delta % above which a `cost_driver` event is marked `critical` instead of `high`. Currently hard-coded to 50%.
4. **Top cost drivers count** — how many top services to emit as `cost_driver` metrics. Currently hard-coded to 5.

## Backend changes

### `backend/config.py`
Add four new optional settings:
```python
AWS_MONTHLY_BUDGET: str = ""                      # numeric dollars; empty = auto
AWS_COST_DELTA_THRESHOLD_PCT: str = "25"          # event creation threshold
AWS_COST_CRITICAL_RISK_THRESHOLD_PCT: str = "50"   # critical severity threshold
AWS_COST_TOP_DRIVERS_COUNT: str = "5"              # number of top drivers
```

### `backend/config_store.py`
Add the new keys to the `aws_cost` defaults dict in `_CONNECTOR_DEFAULTS`.
No secret masking required.

### `backend/mcp/integrations/aws_cost.py`
- Read new config values in `__init__` with safe parsing:
  - `monthly_budget_override`: float or None
  - `delta_threshold_pct`: float (default 25.0)
  - `critical_risk_threshold_pct`: float (default 50.0)
  - `top_drivers_count`: int (default 5)
- In `fetch_metrics`:
  - If `monthly_budget_override` is set and > 0, use it for `monthly_budget`.
  - Use `top_drivers_count` when slicing top services.
- In `fetch_events`:
  - Use `delta_threshold_pct` instead of hard-coded 0.25.
  - Use `critical_risk_threshold_pct` instead of hard-coded 0.5.

### `backend/api/main.py`
Extend the `aws_cost` setup guide in `_GUIDES` with four new fields:
- `AWS_MONTHLY_BUDGET` (type: number, required: false)
- `AWS_COST_DELTA_THRESHOLD_PCT` (type: number, required: false)
- `AWS_COST_CRITICAL_RISK_THRESHOLD_PCT` (type: number, required: false)
- `AWS_COST_TOP_DRIVERS_COUNT` (type: number, required: false)

## Frontend changes

### `frontend/components/ConnectorConfigPanel.tsx`
Add labels for the new keys in the `labels` map (optional, the UI will fall back to the key name otherwise, but labels are cleaner).

### Cost widgets (optional but recommended)
- `frontend/components/widgets/CostSummary.tsx` already renders all `cost_driver` metrics and slices to 6 rows; it will automatically respect the count because the connector emits fewer rows.
- `frontend/components/widgets/CostAnomalyDetection.tsx` already consumes the events generated by the connector; thresholds now come from user config.

No new widget required; the existing pipeline just becomes configurable.

## Validation
- `python -m compileall backend`
- `npm run build`
- Save AWS connector settings in the UI with custom values, re-sync, and verify metrics/events reflect them.

## Why this design
- Reuses the existing connector config layer (`config.py` + `config_store.py` + setup guides) instead of adding a separate settings page.
- Keeps the connector self-contained: it reads its own thresholds from config and emits the right number of metrics/events.
- UI gets the new fields automatically because `ConnectorConfigPanel` renders whatever the backend guide returns.
